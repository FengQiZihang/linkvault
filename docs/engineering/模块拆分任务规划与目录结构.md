## 1. MVP 系统架构

### 1.1 总体结构

LinkVault MVP 采用单体后端 + UniApp 客户端 + MySQL 的简单架构，后续在移动端主链路稳定后再补 Web 前端。

```text
Android APP / H5 调试
        |
        | HTTP JSON / Authorization: Bearer <token>
        v
Spring Boot 后端单体
        |
        | MyBatis-Plus / SQL
        v
MySQL 8.0
```

说明：

- UniApp 负责页面、交互、登录态保存、接口调用、北京时间展示转换
- Spring Boot 负责鉴权、业务规则、数据隔离、URL 导入、元信息解析调度、统一响应
- MySQL 负责持久化用户、验证码、链接、收藏、标签和收藏标签关系
- Android 正式交付以 APK 为目标，H5 只作为开发调试入口
- Web 前端后续基于同一套接口追加实现，不在当前 MVP 首批交付范围内
- MVP 不引入 Redis、消息队列、搜索引擎、对象存储和服务端 refresh token

### 1.2 后端分层

后端保持单模块单体，不拆多 Maven 模块。代码按职责分层：

```text
Controller -> Service -> Mapper -> MySQL
              |
              +-> Component / Util
```

各层职责：

| 层 | 职责 | 禁止事项 |
| --- | --- | --- |
| `controller` | 接收请求、参数校验、获取当前用户、返回 VO | 写复杂业务规则、直接拼 SQL |
| `service` | 编排业务流程、事务控制、用户数据隔离、规则判断 | 返回 Entity 给前端 |
| `mapper` | 数据访问、复杂查询、关系维护 | 承担业务判断 |
| `entity` | 映射数据库表 | 暴露给接口 |
| `dto` | 请求入参 | 混用为数据库对象 |
| `vo` | 接口出参 | 包含内部敏感字段 |
| `config` | 安全、跨域、MyBatis、异步等配置 | 放业务代码 |
| `exception` | 统一异常和错误码 | 各处散落错误响应 |
| `component` | JWT、短信、URL 解析、时间等可复用能力 | 持有页面语义 |

### 1.3 前端分层

前端按 UniApp 页面 + uView Plus 默认组件 + API 封装 + Pinia 轻量状态组织：

```text
pages -> components/uView Plus -> services/api -> Pinia stores -> utils
             |
             +-> static assets
```

各层职责：

| 层 | 职责 |
| --- | --- |
| `pages` | 页面级状态、路由跳转、用户操作流程 |
| `components` | 基于 uView Plus 组合业务组件，如收藏卡片、标签 chip、空状态 |
| `services/api` | 后端接口调用封装 |
| `stores` | Pinia store，保存登录态、当前用户等必要全局状态 |
| `utils` | 时间格式化、URL 提取、提示封装、常量 |
| `static` | 预置头像、图标等静态资源 |

当前前端实现原则：

- 原型负责约束页面结构、字段和交互流程
- 视觉样式优先跟随 uView Plus 默认组件，不因配色和主题问题阻塞页面开发
- 先把请求、状态、页面和业务组件边界拆清，再考虑统一主题或 Web 端适配

### 1.4 核心数据流

#### 登录流

```text
输入手机号 -> 发送验证码 -> 验证码登录
  -> 后端校验验证码
  -> 未注册则创建用户
  -> 返回 JWT + UserVO + isNewUser
  -> 前端保存 token
  -> 新用户进入信息完善页
```

关键边界：

- 验证码只存在服务端
- 前端只保存 token 和必要用户信息
- 除短信与登录外，其余接口必须携带 token

#### 手动导入流

```text
用户粘贴 URL
  -> 前端提取并校验单个 http/https URL
  -> POST /bookmarks/import
  -> 后端规范化 URL 并计算 hash
  -> 创建或复用 lv_link
  -> 创建或复用当前用户 lv_bookmark
  -> 立即返回 bookmarkId
  -> 后端解析元信息并回填 lv_link
  -> 前端进入承接页保存备注和标签
```

关键边界：

- 元信息抓取失败不能阻断保存
- `lv_link.original_url` 只用于排查
- 用户侧展示和打开链接必须使用 `lv_bookmark.original_url`
- 同一用户重复导入同一链接时返回已有收藏，不覆盖备注和标签

#### 标签与搜索流

```text
标签创建 / 置顶 / 重命名 / 合并 / 删除
  -> 维护 lv_tag
  -> 维护 lv_bookmark_tag
  -> 搜索接口按 user_id + keyword + tagIds + untagged 查询
```

关键边界：

- 标签只归属当前用户
- 多标签筛选是 AND 语义
- 未打标签是状态，不是实体标签
- 有关联收藏的标签不能直接删除，必须先合并

## 2. 后端模块拆分

### 2.1 模块清单

| 模块 | 主要职责 | 主要接口 |
| --- | --- | --- |
| 基础工程模块 | 项目骨架、配置、统一响应、异常、错误码、时间处理、跨域 | 无业务接口 |
| 鉴权与用户模块 | 短信验证码、登录、JWT、当前用户、资料更新、统计 | `/auth/*`、`/me/*` |
| 链接导入模块 | URL 校验、规范化、去重、收藏创建、元信息解析触发 | `/bookmarks/import` |
| 收藏模块 | 最近保存、详情、备注更新、删除收藏 | `/bookmarks`、`/bookmarks/{id}` |
| 标签模块 | 标签列表、创建、重命名、置顶、删除、合并 | `/tags/*` |
| 收藏标签关系模块 | 收藏标签全量替换、手动导入承接状态组织保存 | `/bookmarks/{id}/tags`、`/organize` |
| 搜索模块 | 全部收藏、关键词搜索、标签筛选、未打标签筛选 | `/bookmarks/search` |
| 元信息解析模块 | 平台识别、标题/作者/发布时间解析、状态回填 | 内部能力 |

### 2.2 基础工程模块

优先级：第一批。

职责：

- Spring Boot 单模块骨架
- 统一响应结构 `ApiResponse<T>`
- 业务错误码与全局异常处理
- JWT 配置占位
- MyBatis-Plus 配置
- UTC 时间读写和接口输出工具
- 本地配置与 `.env.example`
- 健康检查或基础启动验证接口可选

交付标准：

- 后端能启动
- 能连接 MySQL
- 能返回统一响应
- 目录结构符合本文件约定

### 2.3 鉴权与用户模块

优先级：第一批。

职责：

- 发送短信验证码，开发环境允许固定验证码或控制台输出
- 手机号验证码登录
- 未注册手机号自动创建账号
- JWT 生成、解析、过期处理
- 当前用户上下文注入
- 获取和更新当前用户资料
- 获取当前用户收藏统计

依赖表：

- `lv_user`
- `lv_sms_code`
- `lv_bookmark`
- `lv_tag`
- `lv_bookmark_tag`

注意事项：

- 新用户默认昵称建议为 `用户` + 手机号后四位
- `avatarUrl` 必须校验系统预置头像白名单
- 统计接口只返回三个数字，不做首页聚合

### 2.4 链接导入模块

优先级：第一批。

职责：

- 接收单个 URL
- 校验 `http://` 或 `https://`
- 规范化 URL，计算 `url_hash`
- 创建或复用 `lv_link`
- 创建或复用当前用户 `lv_bookmark`
- 返回 `bookmarkId`、`duplicated`、`LinkMetaVO`、`savedAt`
- 触发元信息解析

依赖表：

- `lv_link`
- `lv_bookmark`

注意事项：

- 短链展开失败不能阻断保存
- `lv_bookmark.original_url` 保存用户本次粘贴的 URL
- 重复导入不覆盖已有收藏的备注、标签和原始 URL

### 2.5 收藏模块

优先级：第一批覆盖查询详情，第二批覆盖编辑和删除。

职责：

- 获取最近保存收藏列表
- 获取收藏详情
- 更新收藏备注
- 删除收藏并同步删除标签关系

依赖表：

- `lv_bookmark`
- `lv_link`
- `lv_bookmark_tag`
- `lv_tag`

注意事项：

- 所有查询必须带当前 `user_id`
- 详情中的 `link.originalUrl` 来自 `lv_bookmark.original_url`
- 删除收藏不删除 `lv_link`

### 2.6 标签模块

优先级：第一批覆盖列表和创建，第二批覆盖管理操作。

职责：

- 获取标签列表，按置顶和普通分组
- 创建标签
- 重命名标签
- 更新置顶状态
- 删除无关联标签
- 合并标签并删除源标签

依赖表：

- `lv_tag`
- `lv_bookmark_tag`

注意事项：

- 标签名同一用户下唯一
- 普通标签按关联收藏数量倒序
- 删除有关联收藏的标签返回冲突
- 合并时需要去重同一收藏已有目标标签的情况

### 2.7 收藏标签关系模块

优先级：第一批。

职责：

- 全量替换某个收藏的标签关系
- 支持手动导入承接状态一次性保存备注和标签
- 校验所有 `tagIds` 都属于当前用户

依赖表：

- `lv_bookmark`
- `lv_tag`
- `lv_bookmark_tag`

注意事项：

- `tagIds=[]` 表示清空标签
- `organize` 同时更新备注和标签，供手动导入承接状态使用
- 操作前必须确认收藏属于当前用户

### 2.8 搜索模块

优先级：第一批。

职责：

- 全部收藏列表
- 关键词搜索
- 单标签筛选
- 多标签筛选
- 未打标签筛选
- 关键词与标签组合查询

依赖表：

- `lv_bookmark`
- `lv_link`
- `lv_tag`
- `lv_bookmark_tag`

注意事项：

- `keyword` 匹配标题和备注
- `tagIds` 多标签为 AND 语义
- `tagIds` 与 `untagged=true` 互斥
- 未传关键词时按 `savedAt` 倒序
- 传关键词时按标题完全匹配、标题部分匹配、备注命中排序

### 2.9 元信息解析模块

优先级：第一批可做最小可用，后续逐步增强。

职责：

- 平台识别：Bilibili、YouTube、X、微信公众号、普通网页、未知
- 标题、发布作者、发布时间解析
- 解析失败时写入兜底字段和状态
- 回填 `lv_link`

依赖表：

- `lv_link`

注意事项：

- 不影响保存主链路
- MVP 不提供手动重试接口
- `title`、`platform`、`publisher` 必须始终有可展示值
- `published_at` 可为空

## 3. 前端模块拆分

### 3.1 页面模块

| 页面 | 职责 | 主要接口 |
| --- | --- | --- |
| 登录页 | 手机号输入、验证码发送、登录 | `/auth/sms-code`、`/auth/login` |
| 首次资料完善页 | 昵称编辑、预置头像选择 | `GET /me`、`PUT /me` |
| 首页 | 统计、置顶标签、最近保存、入口导航 | `/me/stats`、`/tags`、`/bookmarks` |
| 手动导入承接页 | 同一页面内完成 URL 粘贴、保存成功反馈、备注和标签整理 | `/bookmarks/import`、`/tags`、`POST /tags`、`PUT /bookmarks/{id}/organize` |
| 搜索页 | 关键词搜索、结果列表 | `/bookmarks/search` |
| 标签筛选页 | 全部、未打标签、单/多标签筛选 | `/tags`、`/bookmarks/search` |
| 标签管理页 | 创建、置顶、重命名、合并、删除 | `/tags/*` |
| 收藏详情页 | 查看详情、编辑备注、调整标签、删除、打开原链接 | `/bookmarks/{id}`、备注/标签/删除接口 |
| 我的页 | 用户信息、统计、入口 | `/me`、`/me/stats` |
| 编辑资料页 | 编辑昵称和头像 | `GET /me`、`PUT /me` |

### 3.2 前端公共模块

| 模块 | 职责 |
| --- | --- |
| 请求模块 | baseURL、token 注入、统一错误处理、登录失效跳转 |
| 登录态模块 | Pinia 管理 token、UserVO、退出登录 |
| 时间模块 | UTC ISO 字符串转北京时间展示 |
| URL 模块 | 粘贴文本提取单个 URL、格式校验、多 URL 拦截 |
| 标签组件 | 基于 uView Plus 组合标签 chip、置顶标签样式、选择状态 |
| 收藏组件 | 基于 uView Plus 组合收藏列表卡片、空状态、加载状态 |
| 表单组件 | 基于 uView Plus 处理手机号、验证码、备注、昵称等输入体验 |
| 反馈模块 | 优先使用 uView Plus 的 toast、modal、loading 等反馈能力 |

### 3.3 前端状态边界

- Pinia 全局只保存必要登录态和当前用户信息
- 收藏列表、搜索条件、标签选择等默认放在页面局部状态
- 不保存验证码明文
- 不把后端 `msg` 文案作为核心分支判断依据
- 登录态失效时清理 token 并回到登录页

## 5. 开发任务规划

### 5.1 阶段 0：工程初始化

目标：把仓库从文档阶段推进到可运行工程骨架。

任务：

1. 创建 `backend` Spring Boot 项目骨架
2. 创建 `frontend/mobile` UniApp 项目骨架，并接入 uView Plus 与 Pinia
3. 补充根目录 `README.md`
4. 建立 `.env.example` 和本地配置说明
5. 确认 MySQL 初始化脚本可执行

AI 自验：

- 后端能启动到可连接 MySQL
- 前端 H5 调试能启动
- uView Plus 基础组件能正常渲染，Pinia store 能正常初始化

## 6. AI 开发执行规则

### 6.1 每个模块开工前

AI 应先确认：

- 需求是否已有明确来源
- 接口是否在 `接口文档.md` 中存在
- 字段是否在数据库设计和 SQL 中存在
- 页面是否能从原型或需求中找到对应流程

不应直接新增接口、表字段或页面入口；确实需要新增时，先更新对应文档。

### 6.2 每个模块完成后

AI 输出至少包含：

- 变更说明
- 影响范围
- 已执行的本地运行验证
- 建议用户人工复核点

### 6.3 模块间依赖顺序

```text
基础工程
  -> 账号与鉴权
  -> 链接导入与基础收藏
  -> 标签与导入承接状态
  -> 搜索与回找
  -> 标签管理与收藏维护
  -> Android 真机交付
```

不要先做复杂页面和装饰性 UI，再补后端主链路。MVP 优先保证保存、备注、标签、回找闭环可运行。

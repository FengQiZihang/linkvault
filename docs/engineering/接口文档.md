# LinkVault MVP 接口文档

## 0. 文档目标

本文定义 LinkVault MVP 阶段后端 REST API，服务前后端联调、后端 Controller/DTO 编写和后续测试用例设计。

覆盖范围：

- 手机号验证码登录
- 当前用户信息
- 手动粘贴 URL 导入
- 收藏列表、详情、备注、删除
- 标签创建、查询、重命名、合并、删除
- 收藏标签关系维护
- 搜索与筛选

不覆盖范围：

- 管理后台
- refresh token
- 账号密码登录
- 邮箱绑定
- 数据导出
- AI 总结与语义搜索

## 1. 通用约定

### 1.1 基础路径

所有接口统一使用：

```text
/api/v1
```

### 1.2 鉴权

除短信发送、登录接口外，其余接口均需要登录态。

请求头：

```http
Authorization: Bearer <accessToken>
```

MVP 阶段：

- `accessToken` 有效期 `7` 天
- 不实现 refresh token
- 退出登录由前端清理本地 token

### 1.3 时间格式

接口时间字段统一使用 UTC ISO 8601 字符串，统一带 `Z` 后缀。

示例：

```json
"2026-04-21T10:01:00.000Z"
```

说明：

- 数据库使用 `DATETIME(3)`，按 UTC 写入和读取
- 接口层统一转为 UTC ISO 8601
- 接口输出统一补齐毫秒，格式示例为 `2026-04-21T10:01:00.000Z`
- 后端可能部署在美国或其他非中国时区服务器，接口时间不得依赖服务器本地时区
- Android / 前端面向普通中国用户展示时，默认将接口 UTC 时间转换为北京时间 `Asia/Shanghai`
- 排序、筛选、过期判断等逻辑使用 UTC 时间值，不使用北京时间展示字符串
- `publishedAt` 是内容发布时间
- `savedAt` 是用户保存时间
- 两者不能混用

### 1.4 统一响应结构

响应体统一为：

```json
{
  "**code**": 0,
  "msg": "success",
  "data": {}
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | `integer` | 业务错误码，`0` 表示成功，非 `0` 表示失败 |
| `msg` | `string` | 响应说明文案，可直接用于提示，但前端不得仅依赖它做分支判断 |
| `data` | `object/null` | 成功时返回业务数据，失败时通常为 `null` |

成功响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {}
}
```

失败响应：

```json
{
  "code": 40001,
  "msg": "请求参数错误",
  "data": null
}
```

约定说明：

- HTTP 状态码表达协议层结果，例如 `200`、`401`、`404`、`409`
- 响应体中的 `code` 表达业务层结果，例如 `40102 token 已过期`
- 前端判断失败分支时，先看 HTTP 状态码，再看 `code`
- 前端禁止通过匹配 `msg` 文案做核心业务分支
- 后端成功返回统一使用 `code=0`
- 后端失败返回统一使用非 `0` 的业务错误码

### 1.5 分页结构

分页请求参数统一为：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `page` | `integer` | 否 | `1` | 页码，从 1 开始 |
| `pageSize` | `integer` | 否 | `20` | 每页数量 |

分页响应：

```json
{
  "items": [],
  "page": 1,
  "pageSize": 20,
  "total": 100
}
```

### 1.6 HTTP 状态码约定

| HTTP 状态码 | 使用场景 | 说明 |
| --- | --- | --- |
| `200` | 请求处理成功 | 业务成功，响应体 `code=0` |
| `400` | 请求参数不合法 | 参数校验失败、请求体格式错误、必填字段缺失 |
| `401` | 未登录或登录态失效 | 未携带 token、token 非法、token 过期 |
| `403` | 已登录但无权访问 | 访问不属于当前用户的数据 |
| `404` | 资源不存在 | 收藏、标签、用户资源不存在 |
| `409` | 资源冲突 | 如标签重名、重复绑定关系 |
| `429` | 请求过于频繁 | 短信发送过快、短时间内重复提交 |
| `500` | 服务端异常 | 未预期异常、依赖服务异常 |

说明：

- HTTP 状态码用于接口层语义，对齐前端网关、浏览器、抓包工具和日志系统
- 业务错误码用于前端精细化处理和埋点统计
- 同一类 HTTP 状态下允许细分多个业务错误码

### 1.7 业务错误码

| code | 对应 HTTP 状态码 | 说明 | 典型场景 |
| --- | --- | --- | --- |
| `0` | `200` | 成功 | 请求处理成功 |
| `40001` | `400` | 请求参数错误 | 手机号格式错误、分页参数非法、URL 非法 |
| `40101` | `401` | 未登录或 token 无效 | 未携带 token、token 签名错误 |
| `40102` | `401` | token 已过期 | access token 超过有效期 |
| `40301` | `403` | 无权访问当前资源 | 尝试读取或修改他人的收藏/标签 |
| `40401` | `404` | 资源不存在 | `bookmarkId`、`tagId`、`linkId` 对应数据不存在 |
| `40901` | `409` | 资源冲突 | 标签名重复、重复添加同一标签关系、删除有关联收藏的标签 |
| `42901` | `429` | 请求过于频繁 | 短信发送冷却中、短时间重复提交 |
| `50001` | `500` | 服务端异常 | 未捕获异常、第三方依赖失败 |

MVP 阶段补充约定：

- 如无明确特殊场景，接口失败时优先复用以上通用错误码，不急于扩充大量细分码
- 当某类错误需要前端做独立交互时，再新增更细的业务错误码
- 新增错误码时必须同时补充“对应 HTTP 状态码”和“触发条件”

## 2. 公共数据结构

### 2.1 LinkMetaVO

链接元信息展示对象。

```json
{
  "linkId": 1,
  "originalUrl": "https://b23.tv/DCVvWnI",
  "finalUrl": "https://www.bilibili.com/video/BV1EYdhB4EH6",
  "platform": "BILIBILI",
  "publisher": "杜雨说AI",
  "title": "Claude突然强实名认证，你的号被封了吗？",
  "publishedAt": "2026-04-16T08:12:36.000Z",
  "metaStatus": "SUCCESS",
  "metaFetchedAt": "2026-04-21T10:01:00.000Z"
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `linkId` | `number` | 链接 ID |
| `originalUrl` | `string` | 当前收藏记录保存的用户原始输入 URL |
| `finalUrl` | `string/null` | 短链展开后的最终 URL |
| `platform` | `string` | 信息源平台 |
| `publisher` | `string` | 发布作者 |
| `title` | `string` | 标题 |
| `publishedAt` | `string/null` | 内容发布时间 |
| `metaStatus` | `string` | 元信息抓取状态 |
| `metaFetchedAt` | `string/null` | 元信息抓取时间 |

> **注意**：`originalUrl` 必须从 `lv_bookmark.original_url` 读取，不能使用 `lv_link.original_url`。两者含义不同：`lv_link.original_url` 是该链接对象首次入库时任意用户的输入，仅用于排查；`lv_bookmark.original_url` 才是当前用户本次粘贴的原始输入，是详情页展示和打开原链接的唯一依据。

## 3. 账号接口

### 3.1 发送短信验证码

```http
POST /api/v1/auth/sms-code
```

请求体：

```json
{
  "phone": "13800000000"
}
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "expiresIn": 300,
    "cooldownSeconds": 60
  }
}
```

规则：

- 同手机号 `60` 秒内只能发送 `1` 次
- 验证码有效期 `5` 分钟
- 同一验证码最多允许校验失败 `5` 次，超过后本验证码失效，需要重新发送
- 验证码登录成功后立即标记为已使用，已使用验证码不能再次登录
- 响应不暴露手机号是否已注册
- `dev` 环境允许固定验证码或控制台输出

### 3.2 手机号验证码登录

```http
POST /api/v1/auth/login
```

请求体：

```json
{
  "phone": "13800000000",
  "code": "123456"
}
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "accessToken": "jwt-token",
    "expiresIn": 604800,
    "isNewUser": false,
    "user": {
      "userId": 1,
      "phone": "13800000000",
      "nickname": "用户0000",
      "avatarUrl": null
    }
  }
}
```

规则：

- 未注册手机号自动创建账号
- 新账号创建时先生成默认昵称，建议格式为 `用户` + 手机号后四位
- 首次注册后 `isNewUser=true`
- 前端收到 `isNewUser=true` 后进入首次注册信息完善页，调用 `PUT /api/v1/me` 修改昵称；头像可继续为空
- MVP 不区分注册和登录入口

### 3.3 获取当前用户

响应：

## 4. 链接导入与收藏接口

### 4.1 手动导入 URL

```http
POST /api/v1/bookmarks/import
```

请求体：

```json
{
  "url": "https://b23.tv/DCVvWnI"
}
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "bookmarkId": 1,
    "linkId": 1,
    "duplicated": false,
    "link": {
      "linkId": 1,
      "originalUrl": "https://b23.tv/DCVvWnI",
      "finalUrl": null,
      "platform": "UNKNOWN",
      "publisher": "未知作者",
      "title": "未解析标题",
      "publishedAt": null,
      "metaStatus": "PENDING",
      "metaFetchedAt": null
    },
    "savedAt": "2026-04-21T10:02:00.000Z"
  }
}
```

规则：

- 只接受 `http://` 或 `https://` URL
- 先创建 `lv_link` 和 `lv_bookmark`
- `lv_link` 按规范化 URL 复用，`lv_bookmark.original_url` 保存当前用户本次粘贴的原始 URL
- 元信息异步抓取，不阻断保存成功
- 同一用户重复导入同一链接时返回已有收藏，`duplicated=true`
- 重复导入时不自动覆盖已有收藏的 `originalUrl`、备注或标签
- 短链展开失败不能导致保存失败

### 4.2 获取最近保存收藏列表

```http
# 首页最近保存
GET /api/v1/bookmarks?page=1&pageSize=5
```

接口定位：

- 该接口仅用于首页“最近保存”模块
- 只负责返回当前用户最近保存的收藏
- 不承载标签筛选、未打标签筛选、关键词搜索等查询能力
- 首页默认取最近 `5` 条
- 查看全部、标签筛选页和搜索页统一使用 `GET /api/v1/bookmarks/search`

查询参数：

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `page` | `integer` | `1` | 页码 |
| `pageSize` | `integer` | `5` | 每页数量；首页最近保存默认取 `5` 条 |

规则：

- 仅返回当前用户收藏
- 默认按 `savedAt` 倒序
- 不支持 `keyword`、`tagId`、`tagIds`、`untagged` 等筛选参数
- 前端若处于查看全部、标签筛选页、未打标签页、搜索页或多标签筛选状态，应调用 `GET /api/v1/bookmarks/search`

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "items": [
      {
        "bookmarkId": 1,
        "linkId": 1,
        "title": "Claude突然强实名认证，你的号被封了吗？",
        "platform": "BILIBILI",
        "publisher": "杜雨说AI",
        "publishedAt": "2026-04-16T08:12:36.000Z",
        "note": "关注 Claude 账号风控和实名认证变化，后面评估工具选择时参考。",
        "savedAt": "2026-04-21T10:02:00.000Z",
        "tags": [
          {
            "tagId": 1,
            "name": "AI工具"
          }
        ]
      }
    ],
    "page": 1,
    "pageSize": 20,
    "total": 1
  }
}
```

### 4.5 删除收藏

```http
DELETE /api/v1/bookmarks/{bookmarkId}
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": true
}
```

### 4.6 手动重试元信息抓取

```http
POST /api/v1/bookmarks/{bookmarkId}/metadata/retry
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "bookmarkId": 1,
    "linkId": 1,
    "metaStatus": "PENDING"
  }
}
```

规则：

- MVP 可选实现
- 仅将元信息状态重置为 `PENDING` 并触发异步任务
- 不阻断当前收藏使用

```http
DELETE /api/v1/tags/{tagId}
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": true
}
```

规则：

- 仅允许删除无收藏关联的标签，有关联时返回 `40901`
- 有关联收藏的标签必须先合并到其他标签，合并完成后源标签自动删除

### 5.6 合并标签

```http
POST /api/v1/tags/{sourceTagId}/merge
```

请求体：

```json
{
  "targetTagId": 2
}
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "sourceTagId": 3,
    "targetTagId": 2,
    "movedRelationCount": 5,
    "removedDuplicateCount": 1
  }
}
```

规则：

- 合并方向为 `sourceTagId -> targetTagId`
- 源标签合并后物理删除
- 若同一收藏已有关联目标标签，合并时去重
- 源标签不能合并到自身

## 7. 搜索接口

### 7.1 搜索/筛选收藏

```http
# 全部收藏（首页点“全部”、标签筛选页默认状态）
GET /api/v1/bookmarks/search?page=1&pageSize=20

# 单标签筛选（标签管理页点击“查看该标签的所有收藏”）
GET /api/v1/bookmarks/search?tagIds=1&page=1&pageSize=20

# 多标签筛选（标签筛选页选择多个标签）
GET /api/v1/bookmarks/search?tagIds=1,3&page=1&pageSize=20

# 未打标签筛选
GET /api/v1/bookmarks/search?untagged=true&page=1&pageSize=20

# 关键词搜索
GET /api/v1/bookmarks/search?keyword=Claude&page=1&pageSize=20

# 关键词 + 标签筛选
GET /api/v1/bookmarks/search?keyword=Claude&tagIds=1,3&page=1&pageSize=20
```

接口定位：

- 该接口是收藏查询的统一能力，供“标签筛选页”和“搜索页”共同使用
- 标签筛选页偏浏览式筛选：通常不传 `keyword`，通过 `tagIds`、`untagged` 或空条件查找收藏
- 搜索页偏输入式查找：通常传 `keyword`，也可以叠加 `tagIds`
- 页面入口可以不同，但底层查询语义保持一致

查询参数：

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `keyword` | `string` | 无 | 标题/备注关键词；不传表示不按关键词过滤 |
| `tagIds` | `string` | 无 | 标签 ID 列表，逗号分隔；支持单标签和多标签 |
| `untagged` | `boolean` | `false` | 是否只查未打标签收藏 |
| `page` | `integer` | `1` | 页码 |
| `pageSize` | `integer` | `20` | 每页数量 |

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "items": [
      {
        "bookmarkId": 1,
        "linkId": 1,
        "title": "Claude突然强实名认证，你的号被封了吗？",
        "platform": "BILIBILI",
        "publisher": "杜雨说AI",
        "publishedAt": "2026-04-16T08:12:36.000Z",
        "note": "关注 Claude 账号风控和实名认证变化，后面评估工具选择时参考。",
        "savedAt": "2026-04-21T10:02:00.000Z",
        "tags": [
          {
            "tagId": 1,
            "name": "AI工具"
          }
        ]
      }
    ],
    "page": 1,
    "pageSize": 20,
    "total": 1
  }
}
```

规则：

- 不传 `keyword`、`tagIds`、`untagged` 时，返回当前用户全部收藏
- `keyword` 匹配 `title` 和 `note`
- `platform`、`publisher` 仅作为结果展示字段，MVP 不参与关键词匹配
- `tagIds` 多标签采用 `AND` 语义
- `keyword` 和 `tagIds` 叠加时也是 `AND` 关系
- `tagIds` 与 `untagged=true` 互斥，同时传入返回 `40001`
- 仅传 `untagged=true` 时，返回当前用户未打任何标签的收藏
- 不传 `keyword` 时，结果按 `savedAt` 倒序
- 传入 `keyword` 时，排序优先级为：标题完全匹配 > 标题部分匹配 > 仅备注命中；同一命中层级内按 `savedAt` 倒序

## 9. 元信息异步更新说明

导入 URL 后，后端推荐流程：

1. 校验 URL
2. 创建或复用 `lv_link`
3. 创建或复用当前用户的 `lv_bookmark`，新建时写入当前用户粘贴的 `originalUrl`
4. 返回导入成功
5. 异步解析 URL 元信息
6. 回填 `lv_link`
7. 前端通过详情接口或列表刷新看到最新元信息

元信息状态：

| 状态 | 说明 |
| --- | --- |
| `PENDING` | 等待抓取 |
| `SUCCESS` | 解析成功 |
| `PARTIAL` | 部分字段解析成功或使用兜底 |
| `FAILED` | 解析失败 |

前端展示规则：

- `title` 永远展示
- `platform` 永远展示
- `publisher` 永远展示
- `publishedAt` 可为空
- `metaStatus=PENDING` 时可显示“解析中”

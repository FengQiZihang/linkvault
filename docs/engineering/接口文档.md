# LinkVault MVP 接口文档

## 0. 文档目标

本文定义 LinkVault MVP 阶段后端 REST API，服务前后端联调、后端 Controller/DTO 编写和后续测试用例设计。

覆盖范围：

- 手机号验证码登录
- 当前用户信息
- 手动粘贴 URL 导入
- 收藏列表、详情、备注、删除
- 标签创建、查询、重命名、合并、删除
- 收藏标签关系维护
- 搜索与筛选

不覆盖范围：

- 管理后台
- refresh token
- 账号密码登录
- 邮箱绑定
- 数据导出
- AI 总结与语义搜索

## 1. 通用约定

### 1.1 基础路径

所有接口统一使用：

```text
/api/v1
```

### 1.2 鉴权

除短信发送、登录接口外，其余接口均需要登录态。

请求头：

```http
Authorization: Bearer <accessToken>
```

MVP 阶段：

- `accessToken` 有效期 `7` 天
- 不实现 refresh token
- 退出登录由前端清理本地 token

### 1.3 时间格式

接口时间字段统一使用 UTC ISO 8601 字符串，统一带 `Z` 后缀。

示例：

```json
"2026-04-21T10:01:00.000Z"
```

说明：

- 数据库使用 `DATETIME(3)`，按 UTC 写入和读取
- 接口层统一转为 UTC ISO 8601
- `publishedAt` 是内容发布时间
- `savedAt` 是用户保存时间
- 两者不能混用

### 1.4 统一响应结构

成功响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {}
}
```

失败响应：

```json
{
  "code": 40001,
  "msg": "请求参数错误",
  "data": null
}
```

### 1.5 分页结构

分页请求参数统一为：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `page` | `integer` | 否 | `1` | 页码，从 1 开始 |
| `pageSize` | `integer` | 否 | `20` | 每页数量 |

分页响应：

```json
{
  "items": [],
  "page": 1,
  "pageSize": 20,
  "total": 100
}
```

### 1.6 错误码

| code | 说明 |
| --- | --- |
| `0` | 成功 |
| `40001` | 请求参数错误 |
| `40101` | 未登录或 token 无效 |
| `40102` | token 已过期 |
| `40301` | 无权访问当前资源 |
| `40401` | 资源不存在 |
| `40901` | 资源冲突，如重复标签名 |
| `42901` | 请求过于频繁 |
| `50001` | 服务端异常 |

## 3. 账号接口

### 3.1 发送短信验证码

```http
POST /api/v1/auth/sms-code
```

请求体：

```json
{
  "phone": "13800000000"
}
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "expiresIn": 300,
    "cooldownSeconds": 60
  }
}
```

规则：

- 同手机号 `60` 秒内只能发送 `1` 次
- 验证码有效期 `5` 分钟
- 同一验证码最多允许校验失败 `5` 次，超过后本验证码失效，需要重新发送
- 验证码登录成功后立即标记为已使用，已使用验证码不能再次登录
- 响应不暴露手机号是否已注册
- `dev` 环境允许固定验证码或控制台输出

### 3.2 手机号验证码登录

```http
POST /api/v1/auth/login
```

请求体：

```json
{
  "phone": "13800000000",
  "code": "123456"
}
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "accessToken": "jwt-token",
    "expiresIn": 604800,
    "isNewUser": false,
    "user": {
      "userId": 1,
      "phone": "13800000000",
      "nickname": "用户0000",
      "avatarUrl": null
    }
  }
}
```

规则：

- 未注册手机号自动创建账号
- 新账号创建时先生成默认昵称，建议格式为 `用户` + 手机号后四位
- 首次注册后 `isNewUser=true`
- 前端收到 `isNewUser=true` 后进入首次注册信息完善页，调用 `PUT /api/v1/me` 修改昵称；头像可继续为空
- MVP 不区分注册和登录入口

## 4. 链接导入与收藏接口

### 4.1 手动导入 URL

```http
POST /api/v1/bookmarks/import
```

请求体：

```json
{
  "url": "https://b23.tv/DCVvWnI"
}
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "bookmarkId": 1,
    "linkId": 1,
    "duplicated": false,
    "metaStatus": "PENDING",
    "link": {
      "linkId": 1,
      "originalUrl": "https://b23.tv/DCVvWnI",
      "finalUrl": null,
      "platform": "UNKNOWN",
      "publisher": "未知作者",
      "title": "未解析标题",
      "publishedAt": null,
      "metaStatus": "PENDING",
      "metaFetchedAt": null
    },
    "savedAt": "2026-04-21T10:02:00.000Z"
  }
}
```

规则：

- 只接受 `http://` 或 `https://` URL
- 先创建 `lv_link` 和 `lv_bookmark`
- `lv_link` 按规范化 URL 复用，`lv_bookmark.original_url` 保存当前用户本次粘贴的原始 URL
- 元信息异步抓取，不阻断保存成功
- 同一用户重复导入同一链接时返回已有收藏，`duplicated=true`
- 重复导入时不自动覆盖已有收藏的 `originalUrl`、备注或标签
- 短链展开失败不能导致保存失败

### 4.2 获取收藏列表

```http
# 全部收藏
GET /api/v1/bookmarks?page=1&pageSize=20
# 单标签筛选
GET /api/v1/bookmarks?page=1&pageSize=20&tagId=1
# 未打标签筛选
GET /api/v1/bookmarks?page=1&pageSize=20&untagged=true
```

查询参数：

**基础分页（可与任意筛选模式组合）：**

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `page` | `integer` | `1` | 页码 |
| `pageSize` | `integer` | `20` | 每页数量 |

**筛选模式（三选一）：**

| 模式 | 参数 | 说明 |
| --- | --- | --- |
| 全部收藏 | 不传筛选参数 | 返回当前用户全部收藏，按 `savedAt` 倒序 |
| 单标签 | `tagId={id}` | 返回该标签下的收藏 |
| 未打标签 | `untagged=true` | 返回未打任何标签的收藏 |

规则：

- `tagId` 与 `untagged=true` 互斥，同时传入返回 `40001`
- 多标签交集筛选使用搜索接口 `GET /api/v1/bookmarks/search`

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "items": [
      {
        "bookmarkId": 1,
        "linkId": 1,
        "title": "Claude突然强实名认证，你的号被封了吗？",
        "platform": "BILIBILI",
        "publisher": "杜雨说AI",
        "publishedAt": "2026-04-16T08:12:36.000Z",
        "note": "关注 Claude 账号风控和实名认证变化，后面评估工具选择时参考。",
        "savedAt": "2026-04-21T10:02:00.000Z",
        "tags": [
          {
            "tagId": 1,
            "name": "AI工具"
          }
        ]
      }
    ],
    "page": 1,
    "pageSize": 20,
    "total": 1
  }
}
```

- 不阻断当前收藏使用

## 5. 标签接口

### 5.1 获取标签列表

```http
GET /api/v1/tags
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "pinned": [
      {
        "tagId": 1,
        "name": "AI工具",
        "pinned": true,
        "bookmarkCount": 4,
        "createdAt": "2026-04-21T10:02:00.000Z",
        "updatedAt": "2026-04-21T10:10:00.000Z"
      }
    ],
    "normal": [
      {
        "tagId": 3,
        "name": "账号风控",
        "pinned": false,
        "bookmarkCount": 1,
        "createdAt": "2026-04-21T10:02:00.000Z",
        "updatedAt": "2026-04-21T10:10:00.000Z"
      }
    ]
  }
}
```

规则：

- 置顶标签进入 `pinned`
- 普通标签进入 `normal`
- 普通标签按关联收藏数量倒序

### 5.2 创建标签

```http
POST /api/v1/tags
```

请求体：

```json
{
  "name": "自动化",
  "pinned": false
}
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "tagId": 5,
    "name": "自动化",
    "pinned": false,
    "bookmarkCount": 0,
    "createdAt": "2026-04-21T10:02:00.000Z",
    "updatedAt": "2026-04-21T10:02:00.000Z"
  }
}
```

规则：

- 标签名同一用户下唯一
- 标签名最长 `20` 字
- 重名返回 `40901`

### 5.3 更新标签

```http
PUT /api/v1/tags/{tagId}
```

请求体：

```json
{
  "name": "开发效率",
  "pinned": true
}
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "tagId": 2,
    "name": "开发效率",
    "pinned": true,
    "bookmarkCount": 1,
    "createdAt": "2026-04-21T10:02:00.000Z",
    "updatedAt": "2026-04-21T10:10:00.000Z"
  }

```

规则：

- 修改名称时若命中已有标签，前端应引导用户走合并流程
- 后端直接更新接口遇到重名返回 `40901`

### 5.4 删除标签

```http
DELETE /api/v1/tags/{tagId}
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": true
}
```

规则：

- 仅允许删除无收藏关联的标签
- 有收藏关联时返回 `40901`
- 有关联标签必须先走合并流程

### 5.5 合并标签

```http
POST /api/v1/tags/{sourceTagId}/merge
```

请求体：

```json
{
  "targetTagId": 2
}
```

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "sourceTagId": 3,
    "targetTagId": 2,
    "movedRelationCount": 5,
    "removedDuplicateCount": 1
  }
}
```

规则：

- 合并方向为 `sourceTagId -> targetTagId`
- 源标签合并后物理删除
- 若同一收藏已有关联目标标签，合并时去重
- 源标签不能合并到自身

## 7. 搜索接口

### 7.1 搜索收藏

```http
GET /api/v1/bookmarks/search?keyword=Claude&tagIds=1,3&page=1&pageSize=20
```

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `keyword` | `string` | 否 | 标题/备注关键词 |
| `tagIds` | `string` | 否 | 标签 ID 列表，逗号分隔 |
| `untagged` | `boolean` | 否 | 是否只搜索未打标签收藏 |
| `page` | `integer` | 否 | 页码 |
| `pageSize` | `integer` | 否 | 每页数量 |

响应：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "items": [
      {
        "bookmarkId": 1,
        "linkId": 1,
        "title": "Claude突然强实名认证，你的号被封了吗？",
        "platform": "BILIBILI",
        "publisher": "杜雨说AI",
        "publishedAt": "2026-04-16T08:12:36.000Z",
        "note": "关注 Claude 账号风控和实名认证变化，后面评估工具选择时参考。",
        "savedAt": "2026-04-21T10:02:00.000Z",
        "tags": [
          {
            "tagId": 1,
            "name": "AI工具"
          }
        ]
      }
    ],
    "page": 1,
    "pageSize": 20,
    "total": 1
  }
}
```

规则：

- `keyword` 匹配 `title` 和 `note`
- `platform`、`publisher` 仅作为结果展示字段，MVP 不参与关键词匹配
- `tagIds` 多标签采用 `AND` 语义
- `keyword` 和 `tagIds` 叠加时也是 `AND` 关系
- `tagIds` 与 `untagged=true` 互斥，同时传入返回 `40001`
- 排序优先级为：标题完全匹配 > 标题部分匹配 > 仅备注命中；同一命中层级内按 `savedAt` 倒序

```json
    "quickTags": []
  }
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `totalBookmarkCount` | `number` | 当前用户有效收藏数量 |
| `untaggedBookmarkCount` | `number` | 未打标签收藏数量 |
| `recentBookmarks` | `BookmarkListItemVO[]` | 最近保存收藏 |
| `quickTags` | `TagVO[]` | 首页快捷标签 |

说明：

- 该接口可减少首页首屏请求次数
- MVP 若实现成本较高，也可由收藏列表接口和标签接口组合完成

## 9. 元信息异步更新说明

导入 URL 后，后端推荐流程：

1. 校验 URL
2. 创建或复用 `lv_link`
3. 创建或复用当前用户的 `lv_bookmark`，新建时写入当前用户粘贴的 `originalUrl`
4. 返回导入成功
5. 异步解析 URL 元信息
6. 回填 `lv_link`
7. 前端通过详情接口或列表刷新看到最新元信息

元信息状态：

| 状态 | 说明 |
| --- | --- |
| `PENDING` | 等待抓取 |
| `SUCCESS` | 解析成功 |
| `PARTIAL` | 部分字段解析成功或使用兜底 |
| `FAILED` | 解析失败 |

前端展示规则：

- `title` 永远展示
- `platform` 永远展示
- `publisher` 永远展示
- `publishedAt` 可为空
- `metaStatus=PENDING` 时可显示“解析中”

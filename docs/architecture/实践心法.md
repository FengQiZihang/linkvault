## 具体的实践方法
1. **优先使用高能力模型、强执行工具与高权限运行方式**
  - **模型选择**: 优先使用当前能力更强的模型, 并根据任务复杂度调整思考 / 推理强度.
  - **工具选择**: 优先使用稳定、可脚本化的命令行工具.
  - **权限策略**: 在明确了解风险, 且环境可控的前提下, 可通过高权限参数启动工具, 以减少交互确认成本, 提升自动化执行效率。
    ```bash
    # 终端中使用如下参数启动, 赋予工具最高权限
    your-tool --dangerously-bypass-approvals-and-sandbox
    your-tool --dangerously-skip-permissions
    ```

2. **先明确项目需求与目标边界**
- 在正式开发前, 先清晰定义项目目标, 核心功能, 非功能要求, 输入输出边界, 以及阶段性验收标准.
- 尽量把"要做什么"说具体, 而不是只停留在模糊想法层面.
- 定义 MVP(最小可行功能集合), 控制复杂度 + 验证可行性 + 降低AI失控风险.

3. **确定技术栈、开发规范与环境基线**
- 根据项目类型选择合适的技术栈.
- 同步确定代码规范, 目录结构, 命名规则, 接口约定, 提交规范等工程约束.
- 完成开发环境搭建, 确保人和 AI 都在统一, 可复现的环境中工作.

4. **先做系统架构设计，再做模块拆分**
- 从整体上设计系统架构, 明确各层职责, 模块边界, 数据流向与依赖关系.
- 将复杂系统拆分为可独立开发, 可独立测试, 可逐步集成的模块.

5. **启动 AI 进入模块化开发流程**
- 在前述约束明确后, 使用 AI 按模块推进开发.
- 让 AI 围绕既定需求, 架构和规范执行具体工作, 如代码生成 重构 补全 调试 测试与文档编写 而不是在无约束状态下直接"自由发挥".

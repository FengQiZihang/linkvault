LinkVault

个人链接收纳与备注的轻量化移动端应用

汇报人: 风启自航
开发模式: 人机协同 / AI 辅助研发
项目阶段: MVP 验证闭环

为什么需要 LinkVault？

解决我们在手机浏览信息时，“值得保存，但回头找不到”的核心痛点。

日常信息收集的痛点

❌ 信息孤岛：微信、B站、YouTube 各处收藏，缺乏统一归集入口。

❌ 信息堆积：久而久之，收藏夹退化为无序的“吃灰垃圾堆”。

❌ 上下文缺失：回找时只记得“当时的想法/备注”，但平台无法保存。

VS

LinkVault 的解决方案

✅ 一站式归集：手机端快速复制，一键导入各种平台链接。

✅ 上下文备注：强制或引导用户保存时留下备注，锁住记忆线索。

✅ 多维组织：依靠轻量化的“标签 + 备注搜索”随时找回内容。

LinkVault 系统技术栈

前后端分离，依托工业级标准技术架构，提供高性能与多端适配的能力。

后端技术栈 (Spring Boot 3.5) Java 21
  • 核心与构建：Spring Boot 3.5.x、Apache Maven (企业开发基石)
  • 数据库层：MySQL 8.0 (支持 utf8mb4 Emoji 兼容) + MyBatis Plus 3.5.x
  • 安全与鉴权：JWT 无状态鉴权 (结合 Spring Security Filter)
  • 数据强校验：Jakarta Validation (在 DTO 层拦截不合规入参)
  • 异步并发：ThreadPoolTaskExecutor (多线程并发抓取网页)
  • 服务集成：Java 11 HttpClient、YouTube API v3、Aliyun SMS SDK
前端技术栈 (uni-app) Vue 3 / Vite
  • 前端框架：uni-app (基于 Vue 3，支持编译 Android App / H5)
  • 构建工具：Vite (极速热更新构建打包引擎)
  • 组件库：uview-plus (深度定制的高质量移动端组件库)
  • 状态管理：Pinia (轻量级全局状态管理，存储用户态)
  • 网络请求：基于 uni.request 封装 (支持全局拦截、Token 续期)
  • 设计系统：暗岩与琥珀黑金主题配色 (基于 8px 空间系统)

LinkVault 项目亮点

区分技术亮点与功能亮点，兼顾工程规范性与用户实用价值。

技术亮点 (Technical)
1. API 契约驱动开发
基于 OpenAPI 标准制定接口描述，前后端零偏差并行开发与联调。
2. DTO/VO/Entity 严格分层与强校验
全链路数据隔离，利用 Jakarta Validation 注解拦截一切脏数据入库。
3. 高并发异步多源解析爬虫
导入一键即返，后台利用多线程、HttpClient 与 YouTube API 异步填充。
4. 前端 无状态组件 (Stateless) 物理隔离
`components` 仅接 props/抛 emits，不染指网络与状态，极大提升复用度。
功能亮点 (Functional)
1. 一键式跨平台链接归集
支持 B站、YouTube、公众号等多源链接的一键式智能抓取归集。
2. 深度上下文备注 (1000字上限)
支持超长备注，给链接加上保存当下的记忆索引，防范堆积垃圾。
3. 灵活的多维标签分类
支持多对多关系；提供置顶标签、正常标签及“未打标签”快速分组。
4. 基于主观记忆线索的模糊检索
统一搜索能力，支持关键词匹配标题/备注，与标签列表叠作 AND 筛选。

答辩系统演示流程

下面我将开始进行系统核心链路的演示，闭环步骤如下：

1. 验证码快速登录

验证手机号 Mock 开关下的 123456 一键快速登录，新用户自动注册。

2. 复制链接导入

粘贴 B 站或 YouTube 链接。解析中不阻断保存，秒速进入承接页。

3. 整理备注与标签

在承接页记下个性化备注，并批量打上“AI动态”等标签分类。

4. 多维度检索找回

通过首页统计、标签快速筛选、标题与备注关键词完成精准回找。
